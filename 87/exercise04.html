Electronics Production (Feb. 17)

Process

Below you will find my dictation, notes and documentation about the processes I went throught with regards to designing and building this exercise. A link at the bottom of the page will take you to all of the files that I created for this exercise. I used the files, directions and information for then FabISP, a fab-able in-system programmer. This instructions were used throughout the this exercise to complete this project.

Software Installation

I first began the process by installing the Othermill software and the Eagle software onto my computer. I installed the Othermill softwre so I could control the Othermill using my own computer, and I used the Eagle software due to a colleague telling me it was easier to use this software with regards to not needing to do any file conversions or changes to the orginal png file. He sent me a eagle file that was based on a file from a previous Fab Academy student.

One of my colleagues walked me through how to use the Othermill and the settup process for it. I first learned how to change out the bits using the two wrenches, which one is for turning the nut and the other is for keeping the shaft from moving. I than placed a a 1/64 bit into the Othermill. This bit is used for cutting away the copperboard material. The 1/32 bit is used for cutting the outside edge of the board. This will cut the board out. It is important not to drop these bits, they do break very easily. During a transfer of the 1/64 bit, a colleague and I dropped the bit and it became broken.

The Othermill's software is very straight forward. You can go step by step through it very easily. You need to change the tool part you are using along the way and make sure to setup your depth of the platform that the copper board is sticking too. It is important that the bit is in the green safe zome as well as when you move the x axis that it still touches the back of the platform in the back corner to the left. The copperboard is put on the platform using double sided like tap that makes it stick to the metal platform. It is important to choose your trace, outline and etc. You can also load files for cut here. It is important that all of the physical windows are closed. Plus, when you hear the beeping noise you know it is ready to work. You can see this process in the pictures below:

The Othermill software is great. The more I use it I realize it's ability to limit your mistakes. For example, it will always make you aware of the fact if you do not have the right bit by giving you a prompt. Plus, the warning if your are not contacted or you do not have the window closed. Surface Mount Soldering Practice

Before this exercise, I have never done surface mount soldering. I have only done regular soldering. I began this process by watching a how to video on surface mount soldering. This video was extrmely helpful. I began practicing by using a piece copper board, surface mount tweezers, solder, flex and a soldering iron. I used some resistors and began soldering them to the board. I did this by putting flex onto the board and tinning my iron. I put a little bit of soldering on my iron and placed it onto the board where I put the flex. Next, I held the chip with my tweezers and placed it on to the solder, which I reheated with the iron. I then began to stick. After it cooled, I repeated this process to the other side of the resistor. The first time I did it, I used to much solder. The second time I practiced, I used less solder. You can see this in the images below:

I did practice more with more advanced parts or chips. I feel these will be harder. I also want to use some of the others tools.

Cut Board

I cut my board following the step-by-step directions that the Othermill walks you through as well as one of my colleagues helped me with this process. It was an easy process that lead me to creating my board. I had to run the first cut under trace with the 1/64 bit, which took about 3 hours. Next, I ran the outline cut using the 1/32 bit. I went throught the setup process once again as well as I vacumed the work platform. This helps when using the adhesive tape on the back of the copper board. I learned during this process you can change the origin of the board in Othermill by change the view and moving the object. I used the Eagle software to convert the png file of the board I cut out. I loaded the eagle file into the Othermill for the trace and outline cuts.

Electronic Componets

I used solder paste, tweezers, and the Puhui Infrared IC Heater in the process of soldering the componets to my board. I placed all of the smallest parts on first. I did this by putting a small amount of solder paste on each pad with a needle. I than would put the part on to the stick solder paste using tweezers. I did this one piece at a time, so I would not confuse the parts. I put the connectors pins on by cutting them into two sets of three. I took these sets and super glued them back-to-back. Next, I trimmed the connecter pins down to fit the pad sizes. After completely putting all the compents on the board, I placed it gentely into the oven. I pushed F1 and it baked for 7 minutes. After it was done baking, I hit the S button and pulled out the board. I than used a soldering iron to create the SJ1 and SJ2 solder bridges. I than used a multimeter to check different connections on the board to see if my soldering worked. I found that having a magnification tool with a built in lamp was very helpful for seeing the little pieces.

We made another board type out the three offered from the FabLab. We met with Adam Harris, a PhD in electrical engineering and a 2014 Fab Academy graduate, spent time checking over our work and looking at process to provide some help. I learned that the file we are using from the make file for the ISP already has a make hex code in it, so we do not need to make clean. This can cause an issue, so we need to just make hex and than make fuse. You can also possible use make download or make all instead of make fuse in specific times. This allows the chip to set the fuses for timing purposes. If the process is done incorrectly, you ca lock down the chip. Furthermore, we learned that we had our LED flipped the wrong way as well as we need to change our capacitor to the 10 Uf because we used an incorrect capacitor. We were able to get the board to power up because the LED turned on. We also tried using different firmware, such as using the Attiny firmware 1.2.1 instead. We still had no luck to get it to work. I learned more information, but I feel like we are making small steps in progress, which is frustrating. We did email the FabLab community and began getting feedback about using different. Zaric and Firoe Basile provided supprot with regards to identifying the issue with using the wrong driver and firmware. We made changes and still had no results. We kept getting the driver to work with the baord but it would not communicate with it with the device signature = 0x0000. We used the Adafruit Usbtiny ISP programmer as well as we used the driver installation process too. We did this process on a Windows machine. W even used the avrdudess software for windows to check for connections using this gui instead of manual terminal process.

We think that we might be having an issue with a banch of bad hardware, which I have had experience with this in the past as well as a KickStarter company just posted that they got a bad bach of chips from DigaKey. The other theory that we think could be an issue is the OtherMill not playing nice with the Fab module. We asked the community for help with the OtherMill. Neal told us to try using the G Code as well as Jani Ylioja told us to use the Eagle file for the ISP on the Fab Academly page created last year by Jani. The next step is to try this Eagle file from Jani and cut it on the OtherMill as well as surface moutn sodler the parts.

We finally got it to work after multiple days, hours and pain. Soldering and following the clean directions ended up being the issue. I did pure surface mount soldering to find success. I first used resin with a paint brush on the board pads. I than hand soldered each part carefully. My first board did not work because it ripped the copper from the board when making adjustmnets to the jumper pins. I was following the directions from Adam Harris which was based on Zaerc's design. Adam used a regulator in his design instead of ziner diodes. We learned this by analyzing the schematic. Finally, we noticed that the jumper pins on the ISP were thin in comparison to Adam's board as well as it did not create a tight connection to cables. When I was trying out a board and it was not tight with the adafruit programmer cable, so I tried it on Adam's board to compare. I realized Adam's boards had thicker pins which created a better connection, so I used thicker pins on my board that worked.
