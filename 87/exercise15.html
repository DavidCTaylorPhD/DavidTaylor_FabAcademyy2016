Process

Below you will find my dictation, notes and documentation about the processes I went throught with regards to completing the Networking and Communications exercise. Each of the step and processs will be covered in seperate sections. The goal of this exercise was to be able to create a wire or wireless boards that both have processors and communicate. A link at the bottom of the page will take you to all of the files that I created for this exercise.

Preparation

I began reviewing Terence's page and Adam's page as well as Shelga's page. By reviewing these pages as well as the expectations and notes for this week's lesson, it helped prepared me for my work.

I decide to use the concept of my photoresistor board from the Input week. I am using a bridge and a node of the board to connect one another to collect data. The concept will focus on having each board having a phototransitor, which will measure the the amount of light and collect a value based on a threshold, which will show the board that is brighter or dark at a given moment. The data collected from the phototransistor input is sent to the computer, if the value of the threshold of light vs dark is the same one message will appear, but if it is different another message will appear in the serial monitor.

Starting Over

I decide to start over. I am doing this because it gives me the opportunity to start with a clean slate. Tom has offered to help with this process and work with me. He supported me through this processs.

I began this process by using the Arduino IDE to complete the coding. Tom first had me mold this process of networking by doing it with two arduinos. I began this process by using the example code under the section titled Wires in the IDE. I than opened the examples master_writer and slave_reciver. I than upload the master code to a master arduino and the slave code to a slave arduino. I left the slave arduino connected to the USB cord, which was attached to the computer. Next, I began wiring the two arduinos together. I connected the the SCLs together as well as the SDA pins too. Lastly, I connected the GND and the 5V pins of the two boards together.

When I opened the serial monitor of the slave code. I recieved values of the 8 pin communicating back and forth with one another. It is important to color code your wires to the pins because it helps with not getting confused. This process was a success. I learned that the built in pull up resistor in the arduino helps bring up the signal after reading the drop. The images below show this process:

Next, I went throught the process of using my Fabduino modified satshakit board as my slave. The design I am using orginal came from satshatkit. The satshatkit files were the foundation for the changes I made to my board in Eagle. I changed the reset and added a GND pin based on Ian's design. Ian helped me with this process of creating the Fabduino.

I than connected the fabduino to my computer and loaded the slave_recieve sketch to the board. Furthermore, I added a serial write line that would state it is connected ont eh serial monitor. Next, I began connecting the wires from the FTDI to the fabduino slave, which I than connected the other arduino master as well as the fabduino slave to each through a breadboard. I did this due to the fabduino not having a pull up resistor. I connected the SCLs and the SDAs together throught the 5.4 K resistors which was connected to the 5V from the FTDI. These resitors allow for the signal to be pulled back up after it drops for the reading between the two boards communicating with each other. The SCL and SDA allow for the boards to communicate quickly with one another with multiple lines of code and commands. I found success once I opened the serial monitor and seen the information being posted as well as the line of code I added.
